Context

V2 Skews Too Light Across the Board

89 CAB members tested V2 on April 1, 2026. The #1 issue: WTF shade, Face Pencil, and skin tone classification all lean lighter than what testers actually use. Secondary issue: undertone detection biases toward Cool when people are actually Neutral.

The AI's shade recommendations are driven by two things: (1) Claude Vision's skin tone/undertone classification from the selfie, and (2) the shade-data mapping table that converts that classification into product recommendations. Both need adjustment.

Root Cause Analysis

Three Compounding Problems

Problem 1: System Prompt Says "Lean Lighter"

The BORDERLINE CASES section of the Claude Vision prompt explicitly instructs: "When between two shades, lean LIGHTER." This was based on V1 "too dark" complaints, but CAB feedback shows the pendulum swung too far. The FACE PENCIL section also says "ALWAYS favor the lighter option" — double-dipping on the bias.

Problem 2: Face Pencil Ranges Already Shifted 1 Lighter in V2

The shade-data mapping already shifted ALL Face Pencil ranges 1 shade lighter vs V1. Combined with the "lean lighter" prompt guidance, this compounds the error.

Krista Baran: actual FP 2, 3 — recommended 06-07 (4 shades off)
Natasha: actual FP 15 — recommended 08-11 (7 shades off, skin tone misclassified)
Gail Hall: actual FP 08 — recommended 08-11 (ok) but detected as Medium when she's Light-Medium

Problem 3: Cool → Neutral Undertone Bias

~15 testers said undertone was detected as Cool when they're actually Neutral. The prompt doesn't have strong enough guidance on distinguishing Cool from Neutral. Cool requires distinctly pink skin — absence of obvious warmth should default to Neutral, not Cool.

What's Working

Keep These in V3

  • Miracle Balm recommendations are solid — Flushed, Dusty Rose, and Pinched Cheeks appear as both recommended AND actually used by most testers
  • Undertone-aware tint differentiation — the V2 concept of varying MB tint by undertone is validated
  • Golden Hour as universal highlighter — no complaints on highlighter recs
  • WTF shade mappings are correct — the problem is the AI classifies too light, which picks the wrong WTF from the map. Fix the classification and WTF accuracy follows
Proposed V3 Changes

Three Targeted Fixes

Change 1: System Prompt — Flip "Lean Lighter" to "Lean Darker"

Remove "lean LIGHTER" from borderline cases. Replace with: "The most common misclassification is reading someone as lighter than they are. When in doubt, go ONE STEP DARKER." Strengthen Beige default: "Unless the skin is clearly pink-fair with no warmth, default to Beige."

Prompt Changes Detail

  • Borderline Cases: "Lean LIGHTER" → "Lean toward the MORE COMMON shade / one step DARKER"
  • Fair vs Light: Add "If there is any golden or olive cast, go Light. Fair is reserved for clearly pink-cool skin."
  • Face Pencil: "LEAN LIGHTER" → "Match to the detected range. Do NOT bias lighter."
  • Lighting: Add "Front-facing phone cameras with flash or indoor lighting frequently wash out warmth. Compensate by shifting 1 step warmer/darker."

Change 2: Add "Neutral is Default" Undertone Guidance

New section in the system prompt: "Cool means distinctly PINK. Warm means distinctly GOLDEN/YELLOW. If the skin looks neither obviously pink nor obviously golden, it is NEUTRAL, not Cool. Neutral is the most common undertone — default to it unless you see clear pink or golden cues."

Change 3: Widen Face Pencil Ranges Back Darker

V2 shifted all FP ranges 1 shade lighter. CAB shows that was too aggressive. Widen ranges back up based on tester actuals.

WTF Shade Mappings Stay the Same

The WTF shade mapping in the data layer is actually correct (Fair, Light, Beige, Medium, etc.). The problem is that Claude Vision classifies the skin tone too light, which then picks the wrong WTF shade from the correct map. Fixing the system prompt (Change 1) should fix WTF accuracy downstream without touching the data mappings.
