Where the lift compounds

PDP and ATC rates are nearly flat between V4 and Control — the same number of users browse and add to cart. But more of those carts make it to checkout (+2.4%) and more of those checkouts convert (+4.2% CVR). Combined with a +1.3% AOV lift, NRPV climbs +5.5%. This is a quality-of-traffic win, not a volume-of-traffic win — V4's nav is sending more decisive shoppers down the funnel rather than more total clicks.

Step 7

Key learnings

  • The Find-Your-Shade button on its own loses. V1 (FYS button only, no other nav changes) is the worst variant — every metric negative, NRPV −4.8%, ATC −1.9%, checkout-begin −4.8%. Heatmap shows it sends 47% of users back to the homepage and 78% fewer to the Miracle Balm PDP vs Control.
  • The win comes from category prominence + a destination for FYS. V4 (categories on left + FYS) and V3 (search highlight + FYS) both win — adding FYS works only when paired with a clearer category structure. The standalone FYS button creates decision friction; bundled with discovery scaffolding, it adds value.
  • Removing the mega menu is neutral, not positive. V2 (FYS + no mega menu) is a wash (NRPV −1.0%, p2bc 0.42). The mega menu isn't hurting conversion — its removal alone doesn't move the needle.
  • V4's lift is post-PDP, not pre-PDP. PDP rate is flat vs Control. The win compounds at ATC → checkout → order, plus a +22% order lift on full-size Miracle Balm specifically. This looks like a quality-of-intent gain, not a discovery gain — fewer wandering shoppers, more decisive ones.
  • Mobile is excluded from the test by design. < 0.15% of traffic per variant. Read this as a desktop-only test. Mobile nav variants are a separate workstream.
Step 8

Future test hypotheses

Three directly-motivated follow-ons from the V4 win and the V1 failure mode.

01

V4 — Mobile port

The current test isolated desktop. The category-rail-on-left pattern doesn't translate, but the underlying insight (clearer category prominence + FYS destination) might. Test a mobile equivalent: collapse mega menu sub-categories into a sticky category strip with FYS embedded.

Primary metric

Mobile NRPV vs current mobile control

Why it's motivated

Mobile is > 60% of JRB sessions. A +5% NRPV lift on desktop is meaningful, but a comparable mobile lift would be the larger commercial unlock.

02

FYS destination — Miracle Balm vs WTF vs quiz

V1 lost because FYS sent users into ambiguity. Test where FYS routes users: directly to Miracle Balm shade selector vs WTF shade selector vs the existing shade-finder quiz. This isolates which "shade journey" recovers the V1 loss.

Primary metric

FYS-click → ATC rate, Miracle Balm + WTF order share

Why it's motivated

V4's success is partly the FYS button working — but we don't know which destination drove the +22% MB lift. Testing FYS routes deconfounds nav structure from CTA destination.

03

Category rail without FYS

V4 bundles two changes — category rail + FYS button. To isolate which lever drives the win, run a 3-arm: Control vs Categories-only vs Categories+FYS (=V4). If Categories-only matches V4, FYS is window dressing. If it underperforms, FYS is load-bearing.

Primary metric

NRPV — does Categories-only ≥ V4?

Why it's motivated

If the category rail does most of the work, JRB can simplify the nav further. If FYS does, it's a candidate for site-wide deployment beyond the homepage.
