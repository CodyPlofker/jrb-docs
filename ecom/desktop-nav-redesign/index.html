The Single Most Important Finding

JRB is the only brand in our competitive set with a 3-row desktop header (promo bar + utility row + category bar). Every single competitor — from Merit to Glossier to On Running — uses promo bar + 1 nav row at most. JRB's desktop header consumes ~160px of vertical space before any content appears. The average competitor uses ~100px. Combined with the sticky behavior on scroll, this creates the claustrophobic feeling. Desktop users are only 7.5% of sessions but drive 20.1% of purchases — making this the highest-leverage per-visitor surface to optimize.

Universal Patterns (8+ brands)

  • Single nav row — 8 of 10 brands fit all navigation into one row (promo bar excluded)
  • Logo left or center — never right-aligned
  • 5-7 primary items max — brands with more items (Glossier 8, Aesop 10) use tight spacing
  • Utilities right-aligned — Search, Account, Bag always grouped on the right
  • No second category row — JRB is the outlier here

Emerging Patterns Worth Adopting

  • Quiz/Shade Finder as primary nav item — Merit, Saie, Kosas all elevate this. Kosas uses a ✨ icon for visual pop.
  • CTA button in nav — AG1's "Shop All" as a pill button. Creates visual hierarchy without a second row.
  • Visible search bar — Rare Beauty, Mejuri, Aesop show the full search input. Most use icon-only.
  • Transparent nav overlay — Saie, On Running, Mejuri overlay on the hero. More content visible above the fold.
  • "Learn" section elevated — Saie, Drunk Elephant, Aesop all give editorial/education content primary nav real estate.

Key learning: The "Quiz Button" variant hurt conversion — but this doesn't mean quiz in nav is bad. It means the specific execution (adding MORE to an already-crowded nav) backfired. The path is simplification FIRST, then quiz integration.

Bestsellers Link Test

Feb 21-27, 2025 · 1.2M visitors

Adding "Bestsellers" to nav hurt performance: -2.1% CR, -1.4% RPV. ATC rate dropped -1.9% (p2bb 0%).

Key learning: Adding items to nav dilutes click intent. More options ≠ better navigation. Every item you add competes with existing items for attention.

Nav Font + Badges (Bco Test)

Jul 23 – Aug 16, 2024 · 1.6M visitors · 4 variations

"Ringside + Badges" showed +1.3% RPV and +2.1% collection page views (best variant, p2bb 54% — not conclusive but directional). Geogrotesque font change alone was neutral.

Key learning: Visual badges ("NEW", "LIMITED EDITION") on nav items increased collection engagement. Typography alone doesn't move the needle, but badges create urgency cues.

Mobile Nav CTA + EE Nav Redesign (Failed)

Multiple tests, 2024-2025

Mobile CTA in nav was flat (no lift). EE's full navigation redesign was pulled after 1 day — too risky.

Key learning: Full redesigns are too risky to A/B test — the variance is too high. Incremental changes to specific elements (item order, CTA placement, typography cues) are the path.

Test History Synthesis — What We Know Works

Proven Positive
  • Rich mega menu with product imagery (JRB02 winner)
  • Search in mega menu (+3% RPV — JRB05 winner)
  • Visual badges on nav items (directional lift)
  • Condensed/simplified nav (close to control)
Proven Negative
  • Adding more items to an already-crowded nav (Bestsellers -2.1% CR)
  • Adding quiz button without simplifying first (-3.2% CR)
  • Full redesign swaps (EE redesign — pulled immediately)
  • Old nav structure with new dropdown (-4.3% CR)

Recommended Test Plan

Based on JRB's test history showing that incremental > wholesale, and the "Bestsellers" test proving that adding items to a crowded nav hurts:

Phase 1: Simplify First (Week 1-3)

Test Variation A (The Simplifier) or Variation D (The Hybrid) against control. These are the lowest-risk changes — same categories, just collapsed into one row. If this wins, it proves the thesis that less vertical space = better performance.

Phase 2: Quiz Integration (Week 4-6)

With the simplified nav as the new baseline, test Variation B (Quiz-First) — adding the quiz pill button. This isolates the quiz button variable from the simplification variable. If JRB07's quiz button failed because of clutter, it should win in a cleaner nav.

Phase 3: Bold Exploration (Week 7+)

If Phases 1-2 show that simpler + quiz works, test Variation C (The Radical) as a high-risk, high-reward exploration. The transparent overlay + 3-item nav is a significant departure but could unlock major above-the-fold content gains.

Critical Guardrail:

Do NOT test multiple variables at once. The JRB07 test mixed navigation structure changes with quiz button addition and couldn't isolate which variable caused the regression. Test one thing at a time: first simplify, then add quiz, then go radical.
