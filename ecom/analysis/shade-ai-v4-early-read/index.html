Three top-line takeaways

The fix worked. "Too Light" went from 35% (V3) to 6% (V4) — the prompt-contradiction patch and selfie-lighting compensation did what they were supposed to.

Pendulum swung. 34% are now "Too Dark." Mostly Light / Light-Medium customers — the lean-darker override needs a softer touch on the lighter end of the spectrum.

Cool faces reading Neutral. Stricter Neutral default helped overall (+11 pts undertone), but it's now mis-reading fair-pink Cool customers as Neutral.

Lean-darker landed on Light/Light-Med (overshooting) but never reached Medium+ (undershooting). This is a bucket-targeting problem, not a global tuning problem.

V4.1 candidate patches (refined) (1) Conditional lean-darker by undertone: if detected as Neutral, soften +1.5 shades; if Warm, strengthen +1 shade. (2) Loosen Cool detection — currently picked 1/47. Add explicit "fair + visible pink + no warm hues = Cool, not Neutral" rule. (3) Suppress or recalibrate the confidence flag — it's currently inversely correlated with skin-tone accuracy. (4) Add Light × Cool worked example from Robin/Rayna/Kim/Jill miss patterns. (5) Push V4 in dark-tone CX channels — Dark/Deep buckets still have zero data after 47 entries.

The model is more wrong when more sure on skin tone. High-confidence reads are correct 44% of the time vs 55% for medium-confidence. Undertone confidence is meaningless (~tied). Suggests the confidence signal is uncalibrated and shouldn't be surfaced to customers as-is. Could potentially flip it: low-confidence triggers a "try a different lighting" UX prompt.

Other slices worth running once we have more data

Lighting / context from notes. Many customers volunteer "indoor light," "afternoon window," "bathroom lighting." Tag entries by lighting context and see if the bias shifts. Hypothesis: the selfie-lighting comp is calibrated for flash/front-light but over-corrects in flat indoor light.

Repeat user consistency. Mary Dees has 7 attempts across lighting conditions, all "Too Dark." Susan has 3, all "Too Dark." When the same person gets the same wrong answer across lighting, it's the model not the photo. Group entries by name to find systemic vs situational misses.

Confidence × correctness. Claude returns a confidence score on each match. Plot confidence vs accuracy — if low-confidence reads are reliably wrong, we can surface a "we're not sure, try again with better light" UX path.

Detected undertone × shade drift sign. Test the hypothesis that Warm-detected reads are systematically too-light and Neutral-detected reads are systematically too-dark. If true, the undertone classification itself is a leading indicator of skin-tone error and we can use it to triangulate.

Self-reported ethnicity (when filled). When customers volunteer ethnicity in the form, group by that. Helps surface dark-tone gaps before we have submission volume.

Notes sentiment cluster. Quick read on positive ("spot on!" "the closest yet!") vs corrective ("still needs work"). Tells us which buckets feel the AI is "almost there" vs "not even close" — different patches for each.
