Stage 01 — Intelligence

One file. Five inputs. The whole system reads from it.

Customer Intelligence is the foundation. It's a single Markdown file in a private GitHub repo. Five skills write to it. Every other skill — CRO, copywriting, page building — reads from it before doing anything else.

Loom video coming soon

The hub file

One file is the source of truth for everything we know about JRB customers — what they say, what they buy, what they complain about, what they search for, what they wish existed. It lives at:

~/Desktop/01-Projects/jrb-customer-intelligence/customer-intelligence.md Private GitHub repo: CodyPlofker/jrb-customer-intelligence Every team member clones this locally.

Each entry in the file is tagged by source — Prospect, Customer (Behavioral), Customer (Post-Purchase), Customer (Public Reviews), Customer (Survey). That's how downstream skills decide whether they're looking at someone who's never bought, someone who has, someone who returned, or someone who left a five-star review.

Five input skills

Each skill pulls from a different data source, analyzes it, and merges the findings into the shared file with an automatic git pull + git push.

/listenlabs
Qualitative interview transcripts
ListenLabs runs AI-moderated customer interviews. The skill pulls transcripts via API, extracts personas, themes, problems, and golden quotes, then writes the findings to the hub. Twelve studies, four hundred-plus responses on record.
Tool: listenlabs.ai · Tag: Prospect
/outersignal
Purchase-backed behavioral personas
OuterSignal connects to Shopify orders and enriches buyers with demographic, geographic, and professional data. The skill pulls AOV, LTV, repeat rate, and what each persona actually buys.
Tool: outersignal.io · Tag: Customer (Behavioral)
/richpanel
Customer service tickets
Richpanel is our help desk. The skill pulls ticket themes, resolution times, and the actual customer language from messages. Pre-purchase questions become CRO opportunities — the site should answer them.
Tool: richpanel.com · Tag: Customer (Post-Purchase)
/junip
Public product reviews
Junip is the on-site reviews engine. The skill pulls reviews across all SKUs, surfaces ratings, demographic breakdowns, recurring critical issues, and the top customer quotes. The Mini Miracle Balm alone has over ten thousand reviews to mine.
Tool: junip.co · Tag: Customer (Public Reviews)
/jrb-consumer-insights
Typeform survey synthesis
For product launches we run paired Typeform + ListenLabs surveys. The skill takes raw responses and writes them into the GTM brief Consumer Insights section. Then merges the same findings into the hub.
Tool: typeform.com · Tag: Customer (Survey)

Supporting skills

Four more skills work on customer data without writing to the hub.

/simulate-research
Pre-flight a survey question
Reads the hub and simulates how customers would likely answer a strategic question. Quick directional read before commissioning real research. Five-step workflow with four-tier confidence scoring and backtesting against past surveys.
/persona-enrichment
Add psychological depth to personas
Applies Sarah Levinger's Three-Layer System to enrich a persona with belief patterns, identity drivers, and emotional triggers. Used when a persona feels flat and needs the layer below the demographics.
/quick-analysis
Rapid review and feedback analysis
Drop in a CSV of reviews or open-ended survey responses. Get back personas, themes, problems, and golden quotes in minutes. The fastest way to extract structure from unstructured customer language.
/product-review-analysis
Per-product review intelligence
Analyzes Junip reviews for a single SKU. Returns rating distribution, top quotes, critical issues, and demographic breakdowns. Useful when scoping a PDP rebuild.

What reads from the hub

This is where the leverage shows up. Every one of these skills does a git pull on the customer intelligence repo before running, so they always work from the latest data.

  • /jrb-cro — pulls customer language to ground every test idea
  • /jrb-copywriter — pulls customer language for ad copy, email, SMS, landing pages
  • /jrb-video-hooks — pulls golden quotes for scroll-stopping video hooks
  • /newlandingpage — pulls personas and pain points for hero, problem, and proof sections
  • /optimize — pulls customer sentiment to weigh against heatmap and Intelligems data
  • /page-redesign — pulls customer journey signals to prioritize sections in a redesign
  • /ad-iteration-brief — pulls customer language to diagnose what's breaking on a yellow ad

The wiring skill

/ci-connect
Connect a new skill to the hub
When we build a new skill that should read or write to customer intelligence, this skill adds the auto git pull and push steps. Keeps the wiring consistent across the whole skill library.

The reason this works at all: the file is a single source of truth. If five skills each had their own private notes, the system would drift. One file means every skill agrees on who the customer is.

Try it
Run /listenlabs and ask Claude to pull the most recent study and merge findings into the hub. Then run /jrb-cro and ask for a test idea — watch it pick up the new findings without being told.
