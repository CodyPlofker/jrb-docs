Onboarding — 11

Day 1, Week 1, Month 1.

If you're the new Ecom Ops Manager, this is your ramp plan. The framework is deep, but the path through it is short. Most people are productive in a week if they follow this order.

Loom video coming soon

Setup before Day 1

If possible, get this done the day before so Day 1 isn't burned on installs.

  • Install Claude Code (npm install -g @anthropic-ai/claude-code)
  • Install GitHub CLI (brew install gh) and run gh auth login
  • Install the Shopify CLI (brew tap shopify/shopify && brew install shopify-cli)
  • Get GitHub access on the repos in Module 10
  • Get added to the JRB Slack and the #website-changes, #cro, and #socialpaidandorganic channels
  • Get Shopify admin access at jonesroadbeauty.myshopify.com/admin
  • Get tool access — Heatmap, Intelligems, ListenLabs, Richpanel, Junip, OuterSignal, Air, Typeform (URLs in Module 03 and Module 04)

The ramp

Day 1 — Orientation
Read the framework, watch the Looms
  • Watch the Session 01 Loom (Claude Code basics) — link in module 02
  • Read every module of this hub in order, end to end
  • Clone jrb-skills into ~/.claude/skills/
  • Clone jrb-customer-intelligence into ~/Desktop/01-Projects/jrb-customer-intelligence/
  • Clone the EcomExperts theme into ~/Desktop/01-Projects/jrb-theme/
  • Run claude in the terminal. Type /jrb-design-system. Make sure it loads.
Week 1 — Pull data, ship one section
Get hands on the system
  • Run /listenlabs, /junip, /richpanel, and /outersignal end-to-end. See the customer intelligence file fill out.
  • Run /jrb-cro for the current Miracle Balm PDP. Understand how ICE scoring reflects what's already in the hub.
  • Read the existing testing roadmap and the most recent /cro-learnings output.
  • Build one section using /jrb-section-builder. Anything reasonable — practice, not strategy.
  • Run /jrb-qa and /section-audit on it. Fix everything they flag.
  • Pair with Cody for a 30-minute review of the section before pushing anywhere.
Week 2 — Ship end-to-end
First PR to the theme
  • Pick a small change Cody approves — a hero copy edit, a press logo update, a product card adjustment
  • Build it with /jrb-section-builder or by editing an existing cclp-* section
  • Run all three QA gates: /jrb-qa, /section-audit, /ee-audit
  • Convert with /html-to-shopify if needed
  • Push with /jrb-theme-push. Let it ask which preview theme.
  • Open the PR. Get an EcomExperts engineer to review.
  • Watch it merge to develop. That's the first ship.
Month 1 — Own the loop
From contributor to operator
  • Run a full /optimize pass on one underperforming page. Build the test plan, ship the variants in Intelligems, and report results.
  • Take over the daily /page-watchlist alerts in Slack. Triage anomalies, recommend fixes, ship them.
  • Run /cro-learnings at month-end to recalibrate ICE scoring on real outcomes.
  • Find one skill that's missing or weak, and propose an improvement. Push it via /push-skills.
  • Onboard the next person. Update this hub with anything that would have helped you on Day 1.

The default schedule

Once ramped, a typical week looks like this. Adjust to fit your style.

  • Monday. Review weekend Page Watchlist alerts. Read last week's Intelligems test conclusions. Plan the week's tests.
  • Tuesday–Thursday. Build, QA, ship. Keep one CRO test, one section iteration, and one piece of customer intelligence work in flight at any time.
  • Friday. Run /sync-results, write the weekly CRO update, and run /cro-learnings if there's enough new data.

What "good" looks like at 90 days

  • The testing roadmap is self-sustaining — every concluded test loops back, every win compounds into the next test idea
  • The theme has new sections shipping every week, all passing QA on the first pass
  • The customer intelligence file is fresher than it was when you started — at least two new /listenlabs studies and ongoing /junip + /richpanel updates
  • The skill library has at least one improvement you contributed
  • You can run the whole framework without Cody — he reviews PRs and unblocks decisions, you operate the loop

The framework is self-improving. Every test result, every page that wins, every customer interview, every QA failure makes the next pass smarter. The job isn't to memorize it — it's to operate it and keep it learning.

Day 1 first command
Open the terminal. Type claude. Then type /handover and ask "what should I work on today?". The handover skill will read your repos, your recent PRs, and any open Slack threads, and propose a starting point. That's the system meeting you halfway.
