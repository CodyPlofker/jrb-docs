Reference — 10

Where everything lives.

Six GitHub repos hold the framework. Each one has a clear job. Knowing which repo holds what makes the difference between a confused first week and a productive one.

Loom video coming soon

The skills

CodyPlofker/jrb-skills
The skill library — the source of every command in this hub
Every /jrb-* skill referenced anywhere on the site. Each skill is a folder containing a SKILL.md file plus optional references. Push updates with /push-skills. Cloned to ~/.claude/skills/ on every team member's machine.
Public 90+ skills

The data

CodyPlofker/jrb-customer-intelligence
The customer intelligence hub — one shared file
A single Markdown file plus a contributor guide. Every team member clones to ~/Desktop/01-Projects/jrb-customer-intelligence/. Five skills auto-write to it, ten read from it. Auto git pull on read, auto git push on write — keeps the team in sync without thinking.
Private Cloned by every team member

The published outputs

CodyPlofker/jrb-docs
GitHub Pages site for sharable deliverables
Hosts everything that needs a shareable URL — Intelligems test reports, Meta analyses, GTM briefs, presentations. Live at codyplofker.github.io/jrb-docs/. The /publish-doc skill auto-deploys to the right department folder (growth, ecom, product, brand, social, retention, finance, ops).
Public GitHub Pages
CodyPlofker/jrb-landing-pages
Source for standalone landing pages
LPs that live outside the Shopify theme — typically paid-traffic LPs hosted on Vercel. Each LP is its own folder with HTML, assets, and a Paper artboard reference. Sleepless Nights Kit, Mini Extravaganza, Discovery Set LP, and others.
Private Hosted: Vercel per-LP

The theme

EcomExperts JRB theme repo
The Shopify theme — production code
Lives in the EcomExperts GitHub org. Cloned to ~/Desktop/01-Projects/jrb-theme/. Branched from develop, PR'd to develop. Never push to main — that's owned by the EcomExperts deploy pipeline. All Claude-built sections use the cclp- or ccig- prefix.
Private — EcomExperts org PRs target develop
CodyPlofker/jrb-theme-docs
Section registry and theme documentation
Tracks every Claude-built section in claude-created-sections.json and ships a browsable section index at section-audit.html. Updated by /jrb-theme-push as part of the pre-push checklist. The registry is what makes "search before you build" possible — it's how the section library skill knows what already exists.
Private Local: ~/Desktop/01-Projects/jrb-theme-docs/

Side-systems worth knowing

CodyPlofker/jrb-reel-alerts
Example of a standalone production system
A Next.js + Vercel app that monitors JRB Instagram Reels and pings Slack with engagement alerts. Built end-to-end with Claude Code. Useful as a reference for how a small production app gets shipped without leaving the framework.
Public Live: jrb-reel-alerts.vercel.app

How the repos connect

jrb-skills // the toolkit │ ├── reads ──────► jrb-customer-intelligence // the data │ ├── writes to ──► jrb-docs // published outputs │ ├── builds ─────► jrb-landing-pages // LP source │ └── pushes to ──► jrb-theme (EcomExperts) // production theme │ └── tracked in ───► jrb-theme-docs

Access the new hire needs

  • Read access on every public repo above
  • Write access on jrb-skills, jrb-customer-intelligence, jrb-docs, jrb-landing-pages, jrb-theme-docs
  • Contributor access on the EcomExperts theme repo (PR-only, never direct push)
  • GitHub CLI (gh) authenticated locally — Claude uses it to open PRs

One file Cody wishes existed earlier: a jrb-skills CLAUDE.md that explains the skill conventions for new contributors. If you're building a new skill, follow the structure of an existing one — frontmatter, allowed-tools, references folder, output convention. The pattern matters more than the words.
