Stage 03 — Build

Six skills, one rule: search before you build.

The fastest brand-accurate page is the one assembled from sections we've already validated. Six builder skills cover everything from a single section to a full LP redesign — and they all start by checking the section library.

The library comes first

/jrb-section-library
108 sections, 10 components, indexed
Search the existing JRB theme for sections that already do what you need. Press logo bar, FAQ, founder quote, three-step explainer, ingredients accordion — they exist. The library checks before any new section is built.
Allowed tools: Read, Grep, Glob

The cost of skipping this: building a "new" FAQ section that competes with the existing casting-faq.liquid, or a "new" press bar that re-implements jrb11-press-logo-bar.liquid with text logos instead of the real SVGs. Both are common AI failure modes. The library skill prevents them.

The build skills, narrowest to broadest

/jrb-section-builder
A single section, brand-accurate
Builds one section at a time as HTML mockup or Shopify Liquid. Loads the design system automatically. Pulls real images from the Air asset library. Outputs production-ready code that passes QA on the first compile.
Has Air MCP access for real photography
/jrb-page-builder
Full pages, executable build process
Not a reference doc — a step-by-step builder. Walks through each section in order, applies the design system, sources real images, and produces a complete HTML, Liquid, or LP file. Brand-accurate from first render.
/newlandingpage
High-converting LPs from scratch
Combines CRO strategy, brand copywriting, real customer reviews from Junip, and the JRB design system into a single executable workflow. Pulls customer language from the intelligence hub. Output is a paid-traffic-ready LP.
/page-redesign
End-to-end existing-page redesign
Site audit, competitive research across ten-plus competitor pages, customer intelligence pull, then HTML prototypes. Used when an existing page is structurally tired, not just underperforming a section.
/optimize
Full-stack LP optimization
Audits an existing LP with Heatmap data, Intelligems test history, and customer sentiment from the hub. Decides whether to rebuild or iterate. Sizes variants based on traffic. Builds the test plan and executes it.
/section-audit
Pre-build sanity check
Run before building a new section that overlaps with something existing. Compares the proposed section against the JRB design system and existing theme patterns. Catches reinvention.

What "ship" looks like in production

For sections that go into the Shopify theme, see Module 09 (theme push). For standalone landing pages, the source lives in the jrb-landing-pages repo and each LP is deployed to its own Vercel URL. Recent shipped LPs include the Sleepless Nights Kit, Mini Extravaganza, Discovery Set LP, and the Executive First Order. The repo is private, but the live URLs are real.

Where the images come from

Air is JRB's asset library — air.inc. The Air MCP gives Claude direct search access to real product photography, lifestyle images, and brand assets.

mcp__air__search_assets({ query: "miracle balm bottle on cream background", type: "photo" }) // Returns imgix-hosted URLs. // Append ?w=800&h=600&fit=crop&auto=format to resize.

The rule is simple: never use placeholder.com, via.placeholder.com, Unsplash, or gray boxes. If a page goes out with placeholder images, it's a compliance failure.

Real fonts, not Google Font substitutes

Canela, Geogrotesque, BaseMonoWide, RingsideWide, and ArticulatCF load from local files or the JRB Font CDN at jrb-fonts-cdn.vercel.app/fonts/. Not Barlow Condensed. Not Cormorant. Not Inter. The QA skill catches font drift on the way out.

The order of operations

  1. Run /jrb-section-library to check what already exists
  2. If nothing fits, pick the right builder skill (section, page, LP, redesign, optimize)
  3. The skill loads /jrb-design-system + component templates automatically
  4. Real images come in via Air MCP, real fonts via local files
  5. Output is HTML or Liquid, ready for QA
Try it
Run /jrb-section-builder and ask for a "three-step routine explainer for first-time skincare buyers." Watch the design system load, the image search happen, and the section come back ready to drop into a page.
