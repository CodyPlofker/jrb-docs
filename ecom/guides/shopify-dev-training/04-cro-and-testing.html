Stage 02 — CRO

From customer signals to ICE-scored tests, then back again.

CRO is a closed loop. Customer Intelligence and Heatmap signals feed into a prioritized roadmap, the roadmap turns into Intelligems variants, results sync back, and the system recalibrates its own scoring.

The test loop in seven skills

Each skill is one step in the loop. They share data through the customer intelligence file, the testing roadmap, and Heatmap.com.

/jrb-cro
The CRO expert
Generates A/B test recommendations grounded in customer language. Scores each test using ICE — Impact, Confidence, Ease. Has reference docs for awareness levels, pain points, and validated patterns. This is the quarterback for the whole loop.
References: customer-insights.md, validated-patterns.md, ICE rubric
/testing-roadmap
Build a prioritized roadmap
Takes a backlog of test ideas and ranks them with the ICE framework. High Ease equals easy to ship — image swaps score 7-8, structural changes score 4-6. The output is a sequenced roadmap, not a dump of ideas.
/heatmap
Revenue-attributed page intelligence
Pulls clicks, scroll depth, element performance, and device breakdowns from Heatmap.com — but the version that ties every metric to revenue. Eight working endpoints. Every "build a test idea" conversation starts by reading the heatmap.
Tool: heatmap.com · Site ID 2821 · GraphQL API
/intelligems-css
Variant CSS, paste-ready
Generates the CSS that goes into Intelligems for a section-based or sitewide variant. Handles any number of variants, mobile and desktop, and looks up theme selectors automatically. Cuts variant setup from twenty minutes to thirty seconds.
Tool: intelligems.io
/intelligems-analysis
Post-test report
Pulls Intelligems test results, layers in Heatmap.com signals, and produces an HTML report with metrics, device breakdown, product mix, and a recommendation. Auto-publishes to jrb-docs.
Reports site: codyplofker.github.io/jrb-docs
/sync-results
Bulk-sync results back to the roadmap
Pulls every concluded Intelligems test and writes the result onto the matching row in the testing roadmap. The roadmap stays current without manual data entry.
/cro-learnings
Calibrate the system on real outcomes
Reads recent test results, finds patterns ("hero CTA color tests are losing"), and updates ICE scoring weights so future recommendations reflect what actually works at JRB. The system gets smarter with every test.

The watchdog

/page-watchlist
Daily anomaly alerts to Slack
Auto-monitors the top thirty pages plus any manual watches. Detects daily CVR and click-rate shifts versus baseline. When a page drops, it pings #website-changes with the diagnosis. Runs at 8:47 AM every day.

What "ICE" actually means here

  • Impact. Estimated revenue lift if the test wins. Anchored to past similar tests.
  • Confidence. How sure we are the hypothesis is right. Grounded in customer intelligence quotes and prior CRO learnings.
  • Ease. How easy to implement. High Ease equals good — counterintuitive at first. Image swap is 7-8. Full PDP rebuild is 4.

JRB doesn't use Triple Whale. CRO data comes from Shopify Analytics, GA4, and the Heatmap.com revenue-attributed API. Don't recommend tools we don't have wired up.

Why the loop closes

Most CRO programs lose the thread between "we ran a test" and "what we learned." Here, every test result writes back into the roadmap and the calibration. The next test idea is informed by every prior test. Compounding.

Try it
Run /heatmap on the Miracle Balm PDP. Then run /jrb-cro and ask for three test ideas grounded in what the heatmap shows. Watch the ICE scores reflect the customer language already in the hub.
