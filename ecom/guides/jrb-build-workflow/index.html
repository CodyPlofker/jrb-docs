Step 0 Mandatory
Pre-Build Loading
  • Read shared/component-templates.md -- 14 reusable components with exact markup
  • Invoke /jrb-design-system -- load all color, typography, spacing, and button tokens
  • Check /jrb-section-library -- 108 sections indexed; search for reuse before creating new
Step 1 Build
Build the Page
  • Use component templates verbatim for standard sections (star ratings, buy boxes, FAQs, etc.)
  • Use design system tokens for all CSS values -- never hardcode colors, font sizes, or spacing
  • Use Air MCP for real product photography -- never placeholder images
  • Follow section reuse rules: press logos use jrb11-press-logo-bar.liquid, FAQ uses casting-faq.liquid
  • Scope all custom CSS inside #shopify-section-{{ section.id }} to prevent style leaks
  • Apply Canela Light for headlines, Geogrotesque for labels, BaseMonoWide for buttons
Step 2 QA Gate
Post-Build QA
  • Invoke /jrb-qa -- 37+ checklist items across typography, color, buttons, spacing, images
  • Fix ALL Critical issues (wrong fonts, wrong colors, placeholder images, custom button CSS)
  • Fix ALL Warning issues (orphan words, missing hover states, inconsistent spacing)
  • Present to user only after QA passes -- never ship unchecked work
